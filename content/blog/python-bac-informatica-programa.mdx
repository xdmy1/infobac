Python e cea mai grea dintre cele 3 certificări Certiport care echivalează BAC-ul. Nu pentru că Python ar fi un limbaj greu (e contrariul), ci pentru că **examenul testează nuanțe pe care manualele de liceu le sar**.

Articolul ăsta e harta completă a programei. Dacă ești în pregătire activă, folosește-l ca checklist. Dacă încă te uiți de pe margine, citește mai întâi pillar-ul [BAC informatică Moldova 2026: ghidul complet](/blog/bac-informatica-moldova-ghid-2026) ca să prinzi contextul.

## Structura examenului

- **40 de întrebări** multiple choice
- **50 de minute** (≈75 secunde / întrebare)
- **70%** prag de promovare (28/40 corect)
- Limbi: română sau engleză (alegi tu la înscriere)
- **Nu poți reveni** la întrebări (la majoritatea modulelor)

Detalii cu screenshot-uri în [Cum arată un examen Certiport pe viu](/blog/examen-certiport-pas-cu-pas).

## Domeniile examenate (objective domains)

Certiport publică oficial 6 domenii pentru Python. Le-am tradus și grupat aici, cu focus pe ce întreabă realist examenul.

### 1. Operations using Data Types and Operators (≈22%)

Bazic, dar cu capcane:

- Tipuri de date: `int`, `float`, `str`, `bool`, `list`, `tuple`, `dict`, `set`, `None`
- Conversii implicite vs explicite (`int("5")`, `float(3)`, `str(7)`)
- Operatori aritmetici cu prioritate (`**` > `*` > `+`)
- Operatorul de division: `/` (float) vs `//` (integer floor)
- Modulo: `%`
- Comparații: `==`, `!=`, `<`, `>`, `<=`, `>=`
- Operatori logici: `and`, `or`, `not` cu **short-circuit evaluation**
- Slicing: `s[2:5]`, `s[:3]`, `s[-2:]`, `s[::-1]` (reverse)

<Callout tone="warn" title="Capcană clasică">
Întrebări tipice testează diferența dintre `5/2` (=2.5) și `5//2` (=2). De asemenea, comportamentul lui `not` cu valori truthy/falsy: `not 0` = `True`, `not ""` = `True`, `not None` = `True`.
</Callout>

### 2. Flow Control with Decisions and Loops (≈25%)

- `if`, `elif`, `else`
- `while` și `for` (`for x in iterable`)
- `range(start, stop, step)`
- `break`, `continue`, `pass`
- `else` pe loop (rar, dar apare în întrebări)
- Iterare peste dict: `for k in d`, `for v in d.values()`, `for k, v in d.items()`
- Liste ca control de flux (filtrare cu condiție)

```python
# Exemplu tipic de întrebare
total = 0
for i in range(1, 11):
    if i % 2 == 0:
        continue
    total += i
print(total)  # 25 — doar numere impare 1+3+5+7+9
```

### 3. Input and Output Operations (≈10%)

- `input()` — citire de la stdin (string)
- `print()` cu separator (`sep=`) și terminator (`end=`)
- f-strings: `f"Salut, {name}!"`
- Formatare cu `format()` și `%`-style (vechi, dar apare)
- Citire/scriere fișiere:

```python
# Mod text
with open("file.txt", "r") as f:
    content = f.read()

with open("out.txt", "w") as f:
    f.write("Hello")
```

- Moduri: `r`, `w`, `a`, `r+`, `b` (binary)

### 4. Code Documentation and Structure (≈15%)

- Comentarii: `# single line` vs `""" docstring """`
- Indentare (4 spații, **nu** tab — capcană la mixaj)
- Convenții PEP 8 (snake_case, etc.)
- Module: `import`, `from x import y`, `as`
- Module standard: `math`, `random`, `datetime`, `os`, `sys`

### 5. Troubleshooting and Error Handling (≈18%)

- Tipuri de erori: `SyntaxError`, `NameError`, `TypeError`, `ValueError`, `IndexError`, `KeyError`, `ZeroDivisionError`, `FileNotFoundError`
- `try`, `except`, `else`, `finally`
- `raise Exception("...")` și `raise ValueError("...")`
- Excepții multiple: `except (ValueError, TypeError):`
- Custom exceptions (clase derivate din `Exception`)

```python
# Exemplu canonic
try:
    n = int(input("Număr: "))
    result = 100 / n
except ValueError:
    print("Nu e număr.")
except ZeroDivisionError:
    print("Nu împarte la 0.")
else:
    print(f"Rezultat: {result}")
finally:
    print("Gata.")
```

### 6. Operations Using Modules and Tools (≈10%)

- Module standard frecvente: `math.sqrt`, `math.pi`, `random.randint`, `random.choice`, `datetime.now()`, `os.path.exists`, `os.listdir`
- Module third-party (puțin testate, dar apar conceptual)
- `__main__` și `if __name__ == "__main__":`

## Funcții și OOP — partea care omoară

Certiport include **funcții cu argumente default, *args, **kwargs și o introducere ușoară în OOP**. Multe manuale liceene sar peste asta.

### Funcții

```python
def saluta(nume, varsta=18):
    return f"{nume} are {varsta} ani"

def suma(*numere):       # variadic
    return sum(numere)

def info(**kwargs):       # keyword variadic
    for k, v in kwargs.items():
        print(f"{k}={v}")

# Apeluri
saluta("Ion")              # foloseste default 18
saluta("Ion", varsta=20)   # keyword
suma(1, 2, 3, 4)           # 10
info(name="Ion", age=18)
```

### Lambda

```python
patrat = lambda x: x ** 2
print(patrat(5))   # 25

# Cu map/filter/sorted
nums = [1, 2, 3, 4, 5]
pare = list(filter(lambda x: x % 2 == 0, nums))
patrate = list(map(lambda x: x ** 2, nums))
sortat = sorted([(1, "b"), (3, "a"), (2, "c")], key=lambda x: x[1])
```

### OOP minimal

Certiport NU testează OOP avansat. Dar testează:

```python
class Persoana:
    def __init__(self, nume, varsta):
        self.nume = nume
        self.varsta = varsta

    def saluta(self):
        return f"Salut, sunt {self.nume}"

# Moștenire simplă
class Student(Persoana):
    def __init__(self, nume, varsta, scoala):
        super().__init__(nume, varsta)
        self.scoala = scoala

s = Student("Ion", 17, "Mircea Eliade")
print(s.saluta())   # "Salut, sunt Ion"
print(s.scoala)     # "Mircea Eliade"
```

<Callout tone="tip" title="Strategia de pregătire">
Petrece 60% din timp pe **flow control** și **error handling** (împreună ~43% din examen). Petrece 30% pe **funcții/OOP**. Petrece 10% pe **module/IO/docs**.
</Callout>

## Cum să te pregătești în 3 săptămâni

**Săptămâna 1**: tipuri de date, operatori, control de flux. Codeaza zilnic 10 mici probleme (FizzBuzz, factorial, palindrom, etc.).

**Săptămâna 2**: funcții, OOP minimal, fișiere. Fă mini-proiecte (un tracker de note, o bibliotecă mică).

**Săptămâna 3**: simulări full. 40 întrebări sub timp, în fiecare zi. Identifici pattern-urile pe care nu le știi, recapitulezi exact pe alea.

## Greșelile de top la Python Certiport

1. **Confundi `==` cu `=`**.
2. **Modifici un dict / listă în timp ce iterezi peste el** (`RuntimeError`).
3. **Crezi că `is` e identic cu `==`** (`is` testează identitatea obiectului, nu valoarea).
4. **Ignori indentarea** — Python nu iartă.
5. **Nu citești complet enunțul.** Examenul are *capcane verbale* (negații duble).

Detalii în [Greșeli frecvente la BAC informatică (și cum le eviți)](/blog/greseli-frecvente-bac-info).

## Ce nu apare în programa Certiport

Pentru claritate: NU te chinui cu lucruri precum:

- Decoratori (`@property`, `@staticmethod`)
- Generators și `yield`
- Asyncio
- Type hints avansate
- Metaclasses

Toate astea sunt utile pentru programare reală, dar **nu sunt în programa Certiport**. Concentrează-te pe ce e listat mai sus.
